Sets the threshold for the number of new critical, high, medium, or low severity findings allowed. If the number of new findings equals or is greater than the previous builds finding for any one of the severities, the job status will be changed to UNSTABLE or FAILURE.

This setting applies only to synchronous publishing mode!

The previous build is the one that is not aborted and has an analysis result of Dependency-Track, which does not necessarily have to be the immediately previous build.
